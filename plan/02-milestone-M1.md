# M1 — 純觸控完整 Game Loop（精美交付）

> 目標：M1 結束時就是一款**精美、完整可玩**的 2D 對戰遊戲（3D 留 M3）。不略過細節。

## 成功標準（可驗證）
1. `tsc --noEmit` 無錯、`vite build` 通過。
2. 戰鬥引擎（傷害公式、相剋表、STAB、命中）有 vitest 單元測試且全綠。
3. 完整 loop 可玩：選區 → 遇野怪 → 選自有寶可夢（假卡）→ 單招對戰 → 結算。
4. iPad 觸控順手、視覺精緻一致（過場動畫、HP 補間、攻擊演出、傷害數字）。

## 範圍
- **單一遊戲流程**，純觸控，無相機、無 3D、無 QR（QR 在 M2、3D 在 M3、體感在 M4）。
- 本地 seed 資料：約 12 隻種族、各一專屬招式、18 型相剋表、假卡 roster（玩家手牌 3–5 張）。
- 造型：PokéAPI 官方 artwork 圖（runtime 載入，含載入骨架與 fallback）。

## 畫面與演出（精美關鍵）
1. **區域選擇**：主題化區域卡（草原/洞窟/水濱…），各有漸層背景與圖示，點選有縮放/位移過場。
2. **遭遇**：野生寶可夢入場動畫 + 「野生的 ◯◯ 出現了！」字幕；背景依區域。
3. **選卡（載入自有寶可夢）**：玩家手牌以卡牌呈現，選取有翻牌/掃描感動畫（M2 換成真 QR）。
4. **戰鬥**：
   - 經典對戰版面：敵方上方台座、我方下方台座；雙方 HP 條（屬性色、平滑補間）、等級、屬性 badge。
   - **Timing QTE bar**：攻擊時出現來回移動的指針，觸控點擊停下，命中區決定傷害倍率（這是 M4 MediaPipe「握拳停輪盤」的觸控版，走同一 input 抽象介面）。
   - 攻擊演出：施放者前衝/閃光、被擊者震動 + 受擊閃紅、浮動傷害數字、相剋提示（效果絕佳/不太有效）。
   - 回合制：玩家行動 → 敵方行動 → 判定，HP 歸零進結算。
5. **結算**：勝利可投球捕獲（寶貝球晃動 3 下動畫 + 成功/掙脫），失敗顯示再戰；統計本場資訊。

## 技術要點
- **XState** 驅動流程（見 `01-architecture.md` 狀態圖）；React 只 render 當前 state、發 event。
- **Zustand** 管 HP 條、集氣、timing 指針角度等高頻值；**不寫入 React 頂層 state**（為 M3/M4 效能紅線預先養成習慣）。
- **input 抽象層**：`InputSource` 介面回報 `{ chargeLevel, timingHit }`；M1 實作 `TouchInput`，M4 加 `MediaPipeInput`。
- **戰鬥引擎純函數**：`resolveAttack(attacker, defender, move, qte) → { damage, effectiveness, fainted }`，方便單測。
- **framer-motion** 做過場與演出動畫。

## 不做（明確排除，避免 scope creep）
- 多隊伍/換人、PP、異常狀態、道具、經驗值升級、存檔雲端、音樂（音效可選做但不阻塞）。
- 真 3D、QR、相機、MediaPipe。
