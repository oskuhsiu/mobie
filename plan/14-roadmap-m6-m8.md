# 14 — M6~M8 整合路線圖（類型歸類 + 實作順序 + 完整性檢視）

> M6/M7/M8 是分三次設計累積的（延伸系統 wave-1/2、地形模式、技能模組），有不少**跨里程碑、本質相同**的系統。
> 本檔把它們**依類型重新歸類**、給**依賴排序的實作順序**，並做**完整性檢視**確保盤點無遺漏。
> 細節真相仍在各原檔（09–13）；本檔是「怎麼歸類 + 先做什麼」的總控視圖，不重複規格。

## 1. 依類型歸類（找出應合併的同類系統）

> 重點：原本散在 M6/M7/M8 的項目，有 4 組其實是**同一個系統的不同面向**，歸類後合併開發最省工。

### 家族 0 — 共用地基（Foundation，全部依賴它）
| 項目 | 原始 | 作用 |
|------|------|------|
| 掛載地基 + §0.4 回合相位契約 | M6.0 | S1–S8 擴充縫、`resolveTurn(…,{rng,ext})`、settings slice、starStrike→ATTACK mode、相位契約 |
| 模式 contract（arena/wild） | M7.0 | region.mode 玩法 contract、捕獲/地形/意外 gating 集中 setup |
| 技能/場域 schema + `fieldState` 容器 | M8.0 + M8「場域統一」 | SkillDef/ComboDef/Profile/FieldState 型別、catalog、resolveSkillHooks |

### 家族 1 — 戰鬥條件 hook 修飾層（**同一 S1–S8 引擎，三種資料語義**）
> ★ 最大合併點：道具/特性/技能是**同一個 hook 引擎**（圓桌明定「共引擎、分語義」）。建一次引擎，其餘是資料 + 小增補。
| 項目 | 原始 | 語義 |
|------|------|------|
| 持有道具 | M6.b | 可裝可換外掛（建立 S1/S3/S4 + 同步 applyItemTriggers 引擎） |
| 特性 Abilities | M6.g | 種族綁定不可換被動（複用引擎 + onSwitchIn timing） |
| 技能 Skill loadout | M8.a | 可學/換/繼承的條件主動（複用引擎，**不可直接傷害**護欄） |
| 隊伍羈絆 | M6.a | 戰前隊伍級修飾（S2 純函數，同層的「開場一次性」變體） |

### 家族 2 — Combo / Finisher 家族
> ★ 合併點：合體技＝連鎖的特殊變體（圓桌明定）。建連鎖，合體技是它的升級判定。
| 項目 | 原始 | 關係 |
|------|------|------|
| 連鎖攻擊 | M6.d | 多隻依序攻擊（SUBMIT_CHAIN_RESULT） |
| 合體技 + 施放效果 | M8.d | chain variant：2 隻合成大招 + 三類施放效果（灌注地形/全隊增益/敵方弱化） |
| （星擊 finisher） | 已完成 M1.5h | 同家族、能量槽分流（合體技不吃能量、每場一次） |

### 家族 3 — 場域 / 地形層
> ★ 合併點：地形與合體技施放效果共用 `fieldState`（圓桌明定統一容器）。
| 項目 | 原始 | 關係 |

| 合體技灌注地形 | M8.d 一部分 | 灌注 = 寫 fieldState.terrainEffects（接家族 2） |

### 家族 4 — 養成·收集·孵化（成長與 collection）
> ★ 合併點：①進化解鎖技能槽（進化↔技能訓練）②孵化帶技能（孵化↔技能繼承）。

## 2. 建議實作順序（依賴排序；同 Phase 內可並行）

> 原則：地基先行 → 同家族一起做（引擎建一次）→ 依賴在前 → 技能大模組縱向打穿 → 內容橫向鋪。

| Phase | 做什麼 | 為何此時 / 依賴 |
|-------|--------|----------------|
| **P0 地基** | M6.0 掛載地基+相位契約、M7.0 模式 contract | 全部的前提；相位契約是道具/連鎖/技能共守 |
| **P1 hook 層 + 場域地基** | M6.a 羈絆 → M6.b 道具（建 hook 引擎）→ M6.g 特性（複用）；M7.a 地形（**導入 `fieldState` 容器**） | 羈絆最乾淨先驗證 S2；道具建 S1/S3/S4 引擎，特性/技能後面都靠它；地形順帶把 fieldState 容器立起來 |
| **P2 Combo + 地形擴充** | M6.d 連鎖；M7.b 更多/混合/隨機地形 | 連鎖是合體技前置；地形池擴充 |
| **P3 養成·收集·孵化** | M6.c 進化 → M6.f Grade → M6.h 圖鑑成就 → M6.i 孵化 | 進化是技能槽解鎖前置；圖鑑在孵化前（成就→蛋來源）；孵化是技能繼承前置 |
| **P4 模式·長線** | M6.e 連勝塔 → M6.j Ascension；M7.c 野外意外 | 塔依賴模式 contract、且給 SP；Ascension 依賴塔；野外意外依賴 wild 模式 |
| **P5 技能大模組（縱向，小樣本）** | M8.0 schema/fieldState 補全 → M8.a 技能 hook（複用 P1 引擎）→ M8.b 訓練（用 P3 進化槽+P4 塔 SP）→ M8.c 繼承（擴 P3 孵化）→ M8.d 合體技（擴 P2 連鎖+P1 地形灌注）→ M8.e 對手 profile | 圓桌定：**小樣本一次打穿 8.0–e 驗收平衡**；其依賴（道具引擎/連鎖/進化/孵化/塔/地形）在 P1–P4 已就緒 |
| **P6 內容補完（橫向）** | 內容階段 1（G3+天氣地形）→ 2（G4–5+場地地形+混合區）→ 3（G6–9+特殊地形+隨機區） | 引擎驗收後才鋪內容，避免半套資料失真平衡 |

**關鍵排序洞見（合併省工）**：
1. **道具(P1) 一建好 hook 引擎，特性與技能就是「資料 + 小增補」**——不要把三者當三套做。
2. **地形(P1) 一開始就用 `fieldState.terrainEffects`**，別做一次性 `currentTerrains` 再回頭重構——後面合體技施放效果/技能 buff 全靠它。
3. **連鎖(P2) 先於合體技(P5.d)**；**進化(P3) 先於技能槽解鎖(P5.b)**；**孵化(P3) 先於技能繼承(P5.c)**——同家族跨里程碑的依賴要對齊。
4. M8 雖排最後，但它的依賴散在 P1–P4，所以 **P1–P4 設計時就要預留 M8 的接點**（hook 引擎可吃 skill、chain 可升級成 combo、fieldState 可加 status、incubator 可帶 skill、進化可解鎖槽）。

### 檢視時抓到、容易漏的點（已歸位）
- **§0.4 相位契約**：是 M6.0 子項，但屬「跨所有戰鬥系統的契約」→ 歸家族 0，P0 必先落地（道具 S4/連鎖/技能 hook 全共守）。
- **`fieldState` 容器**：橫跨 M7 地形與 M8 場域統一 → 明確歸「**P1 導入容器、P5 補全子欄位**」，避免地形先做成一次性 currentTerrains 後重工。
- **pendingCaptures**（M6.i 子項）、**starStrike→ATTACK mode**（M6.0 子項）、**usedComboKeys**（M8.d 子項）：均為各自父項的內含護欄，隨父項實作。
- **稀有閃光 boss**：在 M7.c 野外意外內，但本質是「對手」→ 與家族 6 對手 profile 同類，實作時可一起考慮 encounter 層 flag。

### 不在本次重整範圍（但記下，避免被當「漏掉」）
- **Backlog（暫不做）**：M6 退回的 12 全域跨場療傷；M7 意外 backlog（暴擊潮/氣象疊加/狂暴化/背水一戰/狙擊先制）；延伸候選未選的（3 場地效果已升 M7、4 狀態/關鍵字、7 隊長覺醒、9 努力值、16 計時連戰、17 每日任務、18…、19/20 網路類）。詳見 09 §7 / 10 §9 / 11 §3。
- **已完成（非新項，僅作家族脈絡）**：星擊(M1.5h)、支援輪盤/球輪盤/連打(M1.5g)——屬 Combo/意外家族的既有成員。
- **其他里程碑**：M4 體感、M5 雲端同步——與 M6–M8 無依賴，獨立。

## 4. 一句話總結
- **歸類**：22 項收斂成 7 個家族；其中 **道具/特性/技能＝同一 hook 引擎**、**連鎖/合體技＝同一 Combo 系統**、**地形/合體施放/fieldState＝同一場域**、**進化↔技能槽 / 孵化↔技能繼承**為跨里程碑合併點。
- **順序**：P0 地基 → P1 hook層+場域地基 → P2 Combo+地形擴充 → P3 養成收集孵化 → P4 模式長線+意外 → P5 技能大模組(縱向小樣本) → P6 內容補完(橫向)。
- **完整性**：22 項全數歸位、無漏；易漏的相位契約/fieldState/子項護欄已點名；backlog 與已完成項明列以免誤判遺漏。
