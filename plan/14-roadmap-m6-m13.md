# 14 — 延伸里程碑路線圖 M6–M13（類型歸類 + 拆成獨立里程碑 + 完整性檢視）

> 原本把 22 項延伸功能擠在 M6/M7/M8 三個里程碑（M6.a–j、M7.a–c、M8.a–e）。
> 既然可拓展里程碑編號，這裡**把每個「家族」升級成一個獨立里程碑（M6–M13）**，依賴排序、各有空間。
> 細節規格仍在原設計檔（09–13）；本檔是「歸類 + 里程碑切分 + 先後順序 + 完整性核對」的總控視圖。
> **舊的 M6.x/M7.x/M8.x 子編號 → 新里程碑**的對應見 §3 表。

---

## 1. 依類型歸類（同類合併，跨原里程碑）

> 4 個「跨里程碑、其實同一系統」的合併點（★）是歸類的核心：

- 家族 **地基**：掛載地基+相位契約、模式 contract、技能/場域 schema。
- 家族 **戰鬥 hook 修飾層** ★：道具 / 特性 / 技能＝**同一 S1–S8 hook 引擎**，建一次、其餘是資料；羈絆是同層的戰前變體。
- 家族 **Combo** ★：連鎖 / 合體技＝**同一 Combo 系統**（合體技是連鎖的升級變體）。
- 家族 **場域/地形** ★：地形 / 合體灌注 / buff＝**共用 `fieldState`**。
- 家族 **養成·收集·孵化** ★：進化↔技能槽解鎖、孵化↔技能繼承。
- 家族 **模式·長線·意外**：連勝塔 / Ascension / 野外意外。
- 家族 **對手**：野外意外的稀有 boss / 對手技能 profile。
- 家族 **內容**：寶可夢補完 / 地形擴充。

---

## 2. 拆成獨立里程碑 M6–M13（依賴排序；每個＝一個家族）

> 與既有 **M4（體感）/ M5（雲端同步）無依賴**，可獨立穿插；下列 M6–M13 是延伸內容的主線順序。

### **M6 — 共用地基（Foundation）**
- 掛載地基：S1–S8 擴充縫 + `resolveTurn(…,{rng,ext})` + settings slice + **§0.4 回合相位契約** + starStrike→ATTACK mode。（原 M6.0）
- 模式 contract：`Region.mode='arena'|'wild'` + gating 集中 setup + 捕獲依 mode + 練習場→競技場。（原 M7.0）
- → 全部後續里程碑的前提。

### **M7 — 戰鬥條件 hook 層（Battle Hook Layer）**
- 隊伍羈絆（S2 純函數，最乾淨先驗證）。（原 M6.a）
- 持有道具（**建立 S1/S3/S4 + 同步 applyItemTriggers 引擎**）。（原 M6.b）
- 特性 Abilities（複用引擎 + onSwitchIn timing）。（原 M6.g）
- → 技能（M12）也複用此引擎；此處把引擎一次建好。

### **M8 — 場域 / 地形（Field & Terrain）**
- 地形效果（terrainMult，**導入 `fieldState` 容器**裝 terrainEffects，別做一次性 currentTerrains）。（原 M7.a）
- 更多 / 混合 / 隨機地形（gen_dex 地形池 + 混合區 + 隨機區）。（原 M7.b）

### **M9 — 連鎖攻擊（Chain / Combo 基底）**
- 連鎖攻擊（連鎖槽 + `chainOpportunity` + `SUBMIT_CHAIN_RESULT` + 連續 QTE overlay + FX）。（原 M6.d）
- → 合體技（M12）是它的升級變體；此處先把連鎖建好。

### **M10 — 養成 · 收集 · 孵化（Progression / Collection）**
- 進化（speciesId 變更；**預留技能槽解鎖接點**給 M12）。（原 M6.c）
- 星級 Grade（派生稀有徽章、零 buff）。（原 M6.f）
- 圖鑑 / 成就（meta 三層語義 + 發獎 action）。（原 M6.h）
- 抽蛋孵化（incubator + pendingCaptures；**預留蛋帶技能接點**給 M12）。（原 M6.i）

### **M11 — 模式 · 長線 · 野外意外（Modes / Replay / Accidents）**
- 連勝塔 / 遠征（RunState + generateRunMap + XState tower；依賴 M6 模式 contract、給 M12 技能 SP）。（原 M6.e）
- 難度修飾 Ascension（塔的二階）。（原 M6.j）
- 野外意外 ×5（wild-only：亂入野生 / 地形突變 / 天降補給 / 稀有閃光 boss / 幸運加碼）。（原 M7.c）

### **M12 — 戰鬥技能大模組（Battle Skill Module；縱向小樣本一次打穿）**
- schema/catalog/persistence + **`fieldState` 補全 4 子欄**。（原 M8.0）
- 技能 hook + loadout（**複用 M7 引擎**，護欄：不可直接傷害）。（原 M8.a）
- 訓練 / 解鎖（SP 買技能 + **M10 進化解鎖槽** + M11 塔給 SP）。（原 M8.b）
- 孵化繼承（**擴 M10 孵化**：蛋帶父母技能）。（原 M8.c）
- 合體技 + 施放效果（**擴 M9 連鎖** + **M8 地形灌注**；usedComboKeys 每場一次）。（原 M8.d）
- 對手 Encounter Skill Profile（純反射 hook、AI 仍只 ATTACK）。（原 M8.e）
- → 圓桌定：**用小樣本（如初代御三家）縱向打穿全部，驗收平衡後**再進 M13。

### **M13 — 內容補完（Content；橫向鋪）**
- 寶可夢 G3→G9 補完（共 1025；PokéAPI 資料 + 官方 artwork runtime URL）。（plan/13）
- 地形擴充（天氣 / 場地 / 特殊型）分階段。（plan/13）

### 檢視時特別點名（易在重整中漏掉者，已歸位）
- **§0.4 相位契約**：M6 子項，但屬跨所有戰鬥系統的契約 → M6 必先落地（道具 S4 / 連鎖 / 技能 hook 共守）。
- **`fieldState` 容器**：橫跨 M8 地形與 M12 技能 → 定「**M8 導入容器、M12 補全子欄位**」，避免地形先做成一次性 currentTerrains 後重工。
- **跨里程碑接點**：進化(M10)↔技能槽(M12)、孵化(M10)↔技能繼承(M12)、連鎖(M9)↔合體技(M12)、地形(M8)↔合體灌注(M12)——**M8–M11 設計時要預留 M12 接點**。
- **子項護欄**：pendingCaptures(M10)、starStrike→ATTACK(M6)、usedComboKeys(M12) 隨父項實作。
- **稀有閃光 boss**：在 M11 野外意外，但本質屬「對手」，與 M12 對手 profile 同類，encounter 層一起考慮。

### 不在本路線範圍（記下，避免被當「漏掉」）
- **Backlog（暫不做）**：12 全域跨場療傷；野外意外 backlog（暴擊潮/氣象疊加/狂暴化/背水一戰/狙擊先制）；未選延伸候選（4 狀態關鍵字 / 7 隊長覺醒 / 9 努力值 / 16 計時連戰 / 17 每日任務 / 19·20 網路類）。詳見 09 §7 / 10 §9 / 11 §3。
- **已完成（家族脈絡，非新項）**：星擊(M1.5h)、支援輪盤/球輪盤/連打(M1.5g)、個體/成長/持久化(M1.5e/f)。
- **獨立里程碑**：M4 體感、M5 雲端同步——與 M6–M13 無依賴，可任意穿插。

- **完整性**：22 項全數歸位、無漏；易漏的相位契約/fieldState/跨里程碑接點已點名；backlog 與已完成項明列以免誤判遺漏。
