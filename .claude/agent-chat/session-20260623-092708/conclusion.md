# 結論：M3 R3F 3D 戰鬥場景 畫面設計審查（claude / gemini / codex）

三方一輪即收斂，無未解分歧。2.5D 大方向（billboard 立繪站發光 3D 地台 + ContactShadows）獲一致認可，保留；本輪只做 P0/P1 微調，不重設整體美術風格。

## 採納的行動清單（高→低）
- P0 底部 safe zone：相機略拉遠+俯角或上抬場景/相機 target，保證我方地台/立繪不壓底部控制列。對手立繪偏大可留（增直式張力），只要不切邊。
- P0 深度線索：地台/圓盤加「透視發光環（橢圓線）或細網格」單一幾何線索撐 3D 透視；不做複雜光帶。
- P1 立繪踩實：每隻立繪腳下加 blob shadow 貼地台；不動 unlit 立繪材質（加光易顯髒，維持乾淨立繪是 2.5D 靈魂）。
- P1 戰鬥易讀性：招式 banner/toast 移到不覆蓋攻擊者/受擊者/命中 FX；受擊 FX 環略縮小貼目標。
- 保留：收服紫地台（當「階段切換」暗示，不必與戰鬥地台統一）、modal 版面、整體 2.5D 方向。

## 可量化驗收標準（codex 提，採用）
- iPad 直式：我方地台最低點 → 控制列上緣 ≥ 24–32px。
- 敵我角色都不切邊。
- 招式 toast 出現時不覆蓋「攻擊者 / 受擊者 / 命中 FX」任一核心區。
- GLB 與 billboard 用同一「視覺高度」基準；寬體測試模型看起來笨重可接受，只要貼地、置中、不卡 UI（06 的測試方塊非 normalize bug——normalize 依高度縮放+置中且有單測）。

## 各方立場差異（已收斂）
- gemini 主張「光照不一致」維持現狀（Low），靠 blob shadow 踩實即可——獲採納。
- codex 額外強調 GLB normalize 與量化驗收，並提醒只做 P0/P1、別順手重設美術——獲採納。
