# 結論 — 寶可夢式多招式制 + Partner(玩家)/Mobie(怪物) 技能切分

四方圓桌（Claude 主持＋參與 / gemini / codex / mistral），round-robin 3 輪達成全體 agree。
主題：本 Mezastar 風 QTE 街機戰鬥（純 reducer、原「每隻單一專屬招」）放寬成寶可夢式多招式制，並把「Partner 玩家技能」與「Mobie 怪物技能」徹底分離。

## 各方立場（收斂前的側重）
- **Claude**：提出整體架構（多招嵌入、buff 下放為變化招、SP 單一經濟、來源映射）。
- **gemini**：UI/街機節奏側重——四鍵/方向映射「選槽即開打」、星擊=身分、變化招走輕量 QTE（反對「單擊確認」會造成節奏冷場）、被動效果**獨立為特性**勿混入招式槽。
- **codex**：安全/契約側重——ATTACK 帶 `slotIndex`＋reducer resolve 後寫 `resolvedMoveId` 進 event；loadout 戰中不可變 snapshot；連鎖規則防爆傷；SP 共用但「分池顯示」護欄；變化招 QTE 只影響幅度不影響成敗。
- **mistral**：實作細節側重——變化招用不同 QTE 模式（按住/連打）視覺區分、效果預覽 UI、對手 AI 招式冷卻防連續重複。

## 共識（完整設計）

### 1. 多招式如何嵌入 QTE 單一行動街機
- **招式槽上限 = 4**（寶可夢式）。每回合：玩家**選 1 招（四鍵/方向映射「選槽即開打」，非 RPG 巢狀選單）→ 立即進入該招 QTE**。靠肌肉記憶維持 Mezastar 流暢打擊感；逾時自動 slot0 / 上回合招。
- **QTE 是「執行品質倍率」，對任何招通用**。
- reducer 改動為 **additive**：`ATTACK` action 帶 **`slotIndex`(0–3)**（忠實反映玩家按了哪槽，防客戶端送未裝備招）；reducer 用 `equippedMoves[slotIndex]` 重驗並 resolve，**把 `resolvedMoveId` 寫進 domain event**（安全＋回放除錯＋M9/M12 連鎖判定皆清楚）。**仍每回合單一 ATTACK action、不開新相位、純 reducer 不破。**
- **戰鬥中 loadout 不可變（snapshot）**；學/忘/換招只在非戰鬥流程。
- **身分（identity）由星擊 finisher 承載**（吃能量槽的單體必殺，維持單一）；多招式只是平時攻防選擇。`species` 既有單一 `moveId` 自然落位成 **slot0「出生自帶招」**＝向後相容。

### 2. Partner(玩家) vs Mobie(怪物) 技能切分 — 核心
- **Mobie(怪物)技能 = 4 招 loadout 本身**：攻擊招 + **變化招**（自我強化/防護/加速/小回復/設地形——即原 M17 規劃的 鼓舞/守護/疾風/回復/整地 buff **下放回招式槽**）。per-creature、走**種族學習表**、出生帶 1、**可學可忘、上限 4**，canonical 存 `OwnedUnit`。← 命中使用者語意「mobie 技能才有上限、才能遺忘」。變化招效果**複用 M7 的 S1/S3/S4 effect 引擎**，只是改由「選招」主動觸發而非被動 hook。
- **Partner(玩家)技能 = 訓練師帳號級工具**：🔍看穿（偵查揭露對手卡）、全隊級訓練師加油/支援輪盤、丟道具。**不綁單一怪物、無 per-creature 上限**、靠玩家養成取得。← 即 M17 瘦身後的純 trainer kit。
- **分界線（全體強調）**：**主動選擇施放 = 招式（佔槽）**；**被動/自動/常駐觸發 = 特性（Ability，M7 已實作，不進招式槽）**。例：劍舞/鐵壁=變化招；威嚇/絕境爆發=特性。
- 兩者**都靠「學習」取得，但來源不同**（怪物=種族學習表；玩家=玩家養成）。

### 3. 招式四來源 → 本遊戲迴圈映射（無大地圖 NPC）
- **領悟(level-up)**：升級自動把學習表到等級的招放進「可學庫」`learnedMoveIds`（自動、免費，主來源）。
- **學習(技能機 TM/教學 tutor)**：合併成「**招式訓練所**」UI，花 **SP** 學該種族 machine/tutor 清單內的招。
- **繼承(蛋招)**：M10 孵化時 egg 帶父母一個蛋招進可學庫（該種族可學者才生效）。
- **出生自帶**：捕獲/孵化依學習表回填現等級以下最靠近的數招進 equipped。
- **招式回憶/遺忘**：同一「招式訓練所」UI——回憶=補回漏學；遺忘=清出招位；loadout 編輯(equipped≤4)亦在此。**全部只在非戰鬥流程。**

### 4. SP 單一經濟（含護欄）
- **SP（玩家技能點，打 boss/塔得）為單一貨幣**，同時供「Mobie 招式訓練」與「Partner 玩家技能」。
- **護欄（codex）**：雖共用來源，但 **UI 與成本曲線分池顯示**，避免玩家誤以為兩者同類。

### 5. 變化招的 QTE 互動（B 題定案）
- 攻擊招走既有命中 QTE。**變化招走輕量 QTE → 決定效果強度**（Perfect/Good/Miss 影響 buff 倍率/回合數）。
- **硬約束**：QTE **只影響幅度/回合數、不影響是否成功**（避免操作失誤把變化招變廢回合、也避免 buff 鏈爆表）；強度有硬上限。
- 可用**不同 QTE 模式**（按住/連打）視覺區分攻擊 vs 變化招，並顯示效果預覽。

### 6. 連鎖(M9)/合體技(M12)與變化招
- **攻擊招**：推進傷害連鎖 / 觸發合體技。
- **變化招**：**不斷鏈**，但貢獻轉化為「連鎖支援值 / buff 強度」（不參與傷害倍率）。
- **合體技門檻**：本次鏈中**至少 1 個攻擊招**，否則只結算輔助合體效果（如全隊回復）。← 杜絕純 buff 鏈爆傷漏洞、又讓輔助配招有價值。

### 7. 對手 AI（守「對手簡單」約束）
- reducer 內**純函式 `chooseOpponentMove(state, rng)`** 加權隨機選 slot（剋制×3、本系×2）；**不新增 AI 相位**。
- 加分項：招式冷卻（用過短暫降權，避免連續重複）、能量滿時提升星擊權重。

### 8. 資料 / 持久化
- canonical 只存 `OwnedUnit.learnedMoveIds`(已學庫) + `equippedMoveIds`(出戰≤4) **兩個 id 陣列**；派生 Move 物件與學習表由種族資料算出、不持久化。
- gen_dex 從 **PokéAPI level-up learnset 降維映射**到精簡 18型×N-tier 招式池，每隻產 `learnset: {level, moveId}[]`(約 3–6 項，跨 tier) + 精簡 machine/tutor 清單；**不吃數百真實招**（街機平衡）。

## 守住的不變式
純 reducer（招式選擇是 slotIndex 注入、resolvedMoveId 由 reducer 算）、單回合單一 ATTACK action 不開新相位、高頻 QTE 走 ref/rAF、canonical 只存兩個 id 陣列、被動效果歸特性不污染招式槽、變化招 QTE 不影響成敗。

## 未解 / 延後（玩測調參，不阻塞）
- 招式槽上限 4 vs 3、SP 曲線（學一招/玩家技能各多少）、變化招強度上限與 QTE 模式具體手感。
- learnset 降維映射的具體規則（每隻取幾招、跨 tier 分佈）需在 gen_dex 實作時定 + 模擬壓力測試平衡。
- 星擊 finisher 與多招式的能量槽互動（是否所有招都蓄能、變化招蓄能率）。
- DQ 魔物作為第二來源時，其「無學習表/特技 vs 呪文」如何套用本招式模型（另案 plan/18 處理）。
