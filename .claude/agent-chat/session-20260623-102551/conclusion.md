# 結論：M2 掃卡/卡庫/產卡 畫面設計審查（claude / gemini / codex）

三方收斂，無未解分歧。整體 M2 畫面方向獲認可；本輪做 P0/P1 微調。

## 採納行動（高→低）
- **P0 產卡 QR「關閉」對比盲區**：ghost 鈕浮在白卡上幾乎不可見＝可用性 bug → 改深色描邊/灰字次要鈕（白底可見）。
- **P1 相機被拒/不支援**：不要留大黑框 → 縮成小提示，且「狀態文案＋手動輸入」收成同一區塊（避免先讀錯誤再找輸入框），手動輸入升為主 CTA。掃卡成功/錯誤蓋在相機框內維持（有「再掃一張」脈絡）。
- **P1 產卡白卡**：保留白底（列印對比），加深色外框/陰影收邊。
- **P1 卡庫列**：文字靠左（目前被 Title 的 `.center` text-align 串成置中，掃清單慢）；刪除低對比＋二次確認（自用資料誤刪成本高）；QR 維持黃主鈕；不做 swipe（省工）。
- **P1 新增自製卡**：拆兩行——搜尋獨佔一行＋候選 chips 在其下；Lv＋異色＋新增下一行。

## 立場差異（已收斂）
- gemini 原提 swipe-to-delete；codex 主張不改 swipe、改低對比＋確認——採 codex（省工、自用情境更安全）。
- 其餘兩方一致。
